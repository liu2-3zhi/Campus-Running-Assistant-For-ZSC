# 跑步助手

> 跑步助手网址：[https://run.zelly.cn](https://run.zelly.cn)

基于 Flask 的 Web 应用，用于模拟校园跑任务的执行，支持多用户、多账号管理，提供 PC 端与移动端双端适配界面。

---

## 使用攻略

### PC 端

#### 1. 注册账号（必须先注册才能登录）

> **温馨提示：**
> 1. **用户名**不能含有中文；
> 2. 注册的**手机号**最好是常用号码，方便后续找回密码；
> 3. 昵称与头像没有要求（可不填 / 不换）；
> 4. **密码**推荐数字 + 字母的高安全性组合（不建议与微信、QQ 等日常密码相同，可记在手机备忘录）；看不清的验证码可点旁边的刷新按钮重新获取。

#### 2. 登录账号（输入注册的账号密码）

> **温馨提示：** 可通过用户名或手机号两种方式登录。忘记密码时可通过手机验证码登录。看不清的验证码可点右边刷新按钮重新获取。

#### 3. 会话管理

> **温馨提示：** 若之前没有会话，直接创建一个新会话（首次默认）。若已有旧会话，请先删除旧会话再开启新会话。

#### 4. 登录校园跑账号

> **温馨提示：** 在单账号登录页面填写校园跑的账号和密码，点击「立即登录」进入相关服务。
>
> 多账号功能默认不开启，如有需要请联系管理员申请。

#### 5. 校园跑流程（重点）

> **温馨提示：**
> 1. 在任务列表中选取需要执行的任务（可通过时间判断）；
> 2. 点击「自动生成路径」（也可在「路径」中手动生成）；
> 3. 点击「开始执行」即可开始校园跑服务。
>
> ⚠️ 校园跑仅在 **08:00 ~ 22:00** 之间可执行，非执行时间段使用后果自负！

#### 6. 其他事项

> **温馨提示：**
> - 新用户首次登录可**免费**使用 2 次校园跑服务，后续每次需支付 **1 元**；
> - 使用过程中遇到问题可在留言板留言，遇到 Bug 可联系管理员，感谢配合！

## 部署

### 使用 Docker Compose（推荐）

```bash
# 克隆项目
git clone https://github.com/liu23zhi/python_runing.git
cd python_runing

# 复制并编辑配置文件
cp config.ini.example config.ini   # 按需修改配置

# 启动服务（HTTP: 8080, HTTPS: 8443）
docker-compose up -d
```

#### 目录挂载说明

| 宿主机路径 | 说明 |
|---|---|
| `./ssl` | SSL 证书目录（只读） |
| `./config.ini` | 主配置文件 |
| `./cache` | 缓存目录 |
| `./logs` | 日志目录 |
| `./school_accounts` | 学校账号数据 |
| `./sessions` | 会话数据 |
| `./payment_orders` | 支付订单数据 |
| `./uploads` | 用户上传文件 |

### 手动安装

```bash
pip install -r requirements.txt
# 或执行安装脚本
bash After_pip_install.sh

python main.py
```
